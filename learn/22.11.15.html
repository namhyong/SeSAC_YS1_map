개발 문화: agile: waterfall model: 가장 익숙한 소프트웨어 개발 기법 고전적인 소프트웨어 생명 주기 병행 수행디지 않고 순차적으로 진행 장점 단순한 모델이라 이해가 쉽다 단꼐별로 정형화된 접근이 가능해 문서화가 가능하다 프로젝트 진행 상황을 한눈에 명확하게 파악 가능하다 단점 변경을 수행하기 어렵다 시스템의 동작을 후반에 가야지만 확인이 가능하다 대형프로젝트에 적용하는 것이 부적합하고, 일저이 지연도리 가능성이 크다. aigle(사상): 에자일이라는 사상을 가지고 스크럼이라는 방법을 사용하는 형태 2주에서 4주정도로 짧은 주기를 잡고 설계, 개발, 테스트 배포 과정을 반복 요구상을 작은 단위로 쪼개 그에 대한 솔루션을 만들고, 빠르게 보여줌으로써 요구 사항에 대한 검증을 진행 scrum(방법론) 2주에서 4주동안 설계, 개발, 테스트 배포를 하는 방법 kanban: to do, doing, done으로 해야할것, 하고 있는 것, 끝낸 것을 나눠서 업무 흐름을 측정과 관리 github에서도 kanban보드 시스템 제공 git: 소스 코드를 효율적으로 관리하기 위해 만들어진 "분산형 버전 관리 시스템" branch: 독립적으로 어떤 작업을 하기 위해 필요한 개념 git branch: 내 로컬 레파지토리에 몇개의 브랜치가 있는지 확인 하는 방법 각각의 가지에서 add,commit,push가 가능 git branch "브랜치 명": 새로운 브랜치 만들기 git checkout "브랜치 명": 다른 브랜치로 이동하기(옮기기) 메인 이외의 다른 브랜치에서 push할때: git push origing 브랜치명 해야 함 -내 로컬에서 브랜치 지우는 법 git branch -d 브랜치 명: 최신에 바뀐것이 메인에 반영되지 않았다고 알림이 온다. git branch -D 브랜치 명 반영하고 싶다면, d를 대문자D로 바꾸면 반영 -github 레파지토리에서 브랜치 지우는 법 git push origin --delete 브랜치 명 git checkout -b "브랜치 명" : 브랜치를 만들고 그 브랜치로 이동하겠다. merge: branch에 branch를 합치는 것 git checkout a git merge b a에 b를 합치는
