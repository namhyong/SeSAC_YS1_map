node.js 구글의 자바스크립트 엔진 java 스크립트를 서버에서 작동할 수 있도록 하는 것 npm 패키지를 통해서 라이브러리 설치하고 관리 가능 runtime: 프로그램 언어가 구동괴는 환경 웹 브라우저 없이 실행 가능(DOM객체는 사용 불가) 특징: 1.자바스크립트 언어 사용 2.Single Thread 에러를 처리하지 못하면 프로그램이 아예 중단됨 => 예외처리의 중요성 주어진 것읋 하나씩 밖에 실행 할 수 없음 3.비동기(non-blocking) I/O 방식 동기: 데이터 요청과 결과가 한자리에서 차례대로 일어남 => 요청을 한개 보내고 결과가 도착해야 다음 요청을 보냄 장점: 설계가 매우 간단하고 직관적이다 단점: 결과가 주어질 때가지 아무것도 못하고 기다리고 있어야 한다. 비동기: 차례로 기다리지 않고 시작 함=> 요청을 한개 보내고 결과가 도착하기 전에 다음 요청을 보냄 장점: 요청에 따른 결과가 반환되는 시간 동안 다른 작업을 수행할 수 있다. 단점: 동기식보다 설계가 복잡하다. 순서를 기다리지 않기 때문에 강제로 순서를 정해줘야 한다. npm: Javascript로 개발된 각종 모듈의 설치, 업데이트, 구성, 제거 과정을 자동화하여 관리해 주는 기능 로컬에서 node.js 설치: https://nodejs.org/dist/v16.18.1/ 접속하여 node-v16.18.1-x64.msi 다운로드 클라우드에서 node.js설치: 푸티에서 순차적으로 입력(붙어넣기는 오른쪽 마우스로 할 수 있음) curl -sL https://deb.nodesource.com/setup_16.x | sudo -E bash - apt-get install nodejs apt-get install npm 버전 확인 방법(로컬:cmd, 클라우드:putty에 입력): node -v npm -v https://nodejs.org/dist/v16.18.1/
